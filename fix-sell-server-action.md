このリポジトリに対して、以下を必ず順番通りに実行してください。

################################
目的
################################

- Sell 機能を「UI 完全保持 + Server Action 完全分離」で安定させる
- Figma UI を一切崩さない
- ログイン → 出品遷移の退化を完全に防ぐ

################################
前提（絶対厳守）
################################

- Figma 由来の UI / className / JSX 構造は変更禁止
- UI の高さ・余白・配置は一切変えない
- 仕様を勝手に追加しない
- 新しいページ・新しい機能を作らない

################################
やること 1：SellForm を UI 専用にする
################################

- src/components/SellForm.tsx を UI 専用コンポーネントにする
- useState / useEffect / client-side submit をすべて削除
- <form action={...}> のみを使う
- props は一切増やさない

################################
やること 2：Server Action を page.tsx に集約
################################

- app/sell/page.tsx に Server Action を定義する
- createItem(formData) を呼び出す
- 成功後は redirect('/') する
- SellForm には Server Action を action として渡す

################################
やること 3：認証ガードの正常化
################################

- page.tsx で supabase.auth.getUser() を使用
- 未ログイン時のみ redirect('/login')
- ログイン済みユーザーは必ず sell ページに入れる
- 二重チェックや例外 throw は削除

################################
やること 4：Header の見た目を Figma 基準に戻す
################################

- Header の高さ・padding・アイコンサイズを調整
- 見た目のみ修正、挙動は変更禁止
- プロフィールアイコンは表示のみ

################################
完了条件
################################

- npm run build が error / warning なしで成功
- ログイン → 出品 → 完了 → トップ遷移が通る
- Figma デザインと完全一致

################################
出力形式
################################

- 変更したファイル一覧
- 各ファイルの全文（省略禁止）
