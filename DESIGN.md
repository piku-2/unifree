# DESIGN

## 目的

本ドキュメントは、本プロジェクトの **UI / UX の方向性** を示すものである。

- 実装対象の確定
- 機能追加の判断
- 実装可否の決定

は **行わない**。

実際に実装してよい UI / 機能は
**必ず `TASKS.md` に記載されたものに限定** する。

---

## デザイン方針

### 基本思想

- シンプルで迷わない UI
- 学内利用を前提とした安心感
- 情報量は最小限

### カラートーン

- ベース：白 / 薄いグレー
- アクセント：落ち着いたアクセントカラー
- 警告・危険操作は明確に視認可能にする

---

## レイアウト指針

### 一覧画面

- 商品カード形式
- 重要情報（画像・価格・タイトル）を優先表示

### 詳細画面

- 商品画像
- 説明
- 出品者情報
- 行動ボタン（購入 / チャット）

### マイページ

- プロフィール情報
- 出品履歴の一覧

---

## 禁止事項（重要）

以下の内容は **本 DESIGN.md に記載されていても実装してはならない**：

- TASKS.md に存在しない機能
- 画像複数枚アップロード前提の UI
- Draft / Confirm などの多段フロー
- お気に入り / Like 機能
- ランキング / レコメンド要素
