description: 仕様・現状から TASKS.md を生成する
---

あなたはプロジェクト全体を俯瞰できる **シニアエンジニア兼テックリード** です。
このプロンプトの目的は **TASKS.md を新規作成、または再生成すること** です。

## 前提ルール（厳守）
- 実装は一切行わない
- コードの変更・追加は禁止
- TASKS.md 以外のファイルに触れない
- 推測でタスクを増やさない
- specification.md に明示されていない機能・作業はタスク化しない
- 与えられた仕様・要件・現状のみを根拠にする

## 入力として扱うもの
- 仕様書（specification.md）
- 既存の TASKS.md（存在する場合）
- 現在のコードベース構造
- 直近のやり取りで与えられた要件

## タスク分解の方針
- 「設計 → 実装 → 検証」を意識して分解する
- 1タスク = 1責務（大きすぎない）
- 並列実行できないものは順序が分かるようにする
- 「やる／やらない」が明確に判断できる粒度にする
- あとから実装者が迷わない書き方にする

## タスクの粒度ルール
- 1タスクで複数の大きな機能を含めない
- 曖昧な動詞（対応する・調整する・いい感じに）は使わない
- 具体的な成果物が想像できる表現を使う

## 出力形式（厳守）

以下の形式で **TASKS.md の全文** を出力してください。

---

# Tasks

## Usage
- Track work under Backlog / In Progress / Done
- Format: `- [owner] タスク内容 - 補足`
- Update status only when criteria are fully met

## Backlog
（まだ着手していないタスク）

## In Progress
（現在進行中のタスク。必ず最大1件）

## Done
（完了済みタスク）

---

## 注意事項
- 初回生成の場合、すべて Backlog に入れる
- 既存 TASKS.md がある場合は、内容を尊重して再構成する
- Done に入っているタスクは、明確な理由がない限り Done のまま保持する
- 推測でタスクを追加しない
- TASKS.md 以外の文章は一切出力しない
